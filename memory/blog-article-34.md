# 技术自由职业者如何报价：把模糊需求变成可成交提案

> 目标平台：掘金 / 知乎 / V2EX / 公众号

> 配套项目：Proposal Generator

---

很多技术人接自由职业单子时，最难的不是写代码，而是报价。

客户常说："能不能帮我做个自动看竞品价格的东西？每天提醒我一下。"

这句话背后可能是一个 ¥800 的脚本，也可能是一个 ¥30,000 的监控系统。区别不在技术栈，而在范围：目标网站数量、是否登录、是否反爬、是否要数据库、是否要告警、是否多人使用、是否长期维护。

所以自由职业者必须掌握一个核心能力：**把模糊需求翻译成可报价的交付范围。**

---

## 报价失控的 3 个原因

1. 客户说的是目标，不是需求。  
2. 技术人容易按"代码量"报价，而不是按业务结果报价。  
3. 没有边界，就没有利润，"顺便"会吞掉所有利润。

---

## 三档报价框架

### 基础版：验证需求
- 单一流程
- 少量数据源
- 手动或半自动运行
- 简单文档
- 不承诺复杂异常处理

参考：自动化脚本 ¥800-2,000；数据采集 ¥1,500-3,000；AI Bot Demo ¥3,000-8,000。

### 标准版：可稳定使用
- 定时任务
- 日志和异常处理
- 数据库或结构化存储
- 部署文档
- 基础告警

参考：竞品监控 ¥5,000-12,000；AI Agent ¥12,000-30,000；企业知识库 ¥15,000-30,000。

### 高级版：业务系统化
- Web 管理后台
- 多用户/权限
- 监控告警
- 审计日志
- 私有化部署
- 长期维护

参考：企业 Agent 平台 ¥50,000+；企业 MCP 网关 ¥50,000+。

---

## 报价前必须问的问题

1. 数据来源：网页、Excel、API 还是数据库？是否登录？是否验证码？  
2. 运行频率：手动、每天、每小时还是实时？  
3. 交付形态：脚本、Web 页面、API、Docker 还是云部署？  
4. 权限安全：是否访问内部系统？是否允许写操作？是否要审计？  
5. 维护方式：上线后谁维护？服务器/API 成本谁承担？

## 客户话术模板

你好，我看了你的需求，初步判断这是一个「数据采集 / 竞品监控」项目。建议先做标准版，预算大概在 ¥5,000 - ¥12,000，周期约 1-2 周。交付内容包括多站点采集、定时任务、数据库存储、降价/变化提醒。开工前我需要确认：目标网站 URL、采集字段、每天数据量、是否需要登录或代理。如果范围确认，我可以给你出详细执行方案和里程碑。

## 配套工具：Proposal Generator

我做了一个零依赖 Python 工具，可以根据一句客户需求，自动输出：

- 项目类型判断
- 推荐报价档位
- 三档价格范围
- 交付物清单
- 开工前问题
- 风险边界
- 可直接发给客户的话术

```bash
python proposal_generator.py "帮我做一个自动抓取竞品价格并每天发微信提醒的工具"
```

示例输出：

```text

预计周期：1-2 周
```

技术自由职业者想赚到钱，不能只会写代码，还要会定义范围、表达价值、控制风险。

记住三句话：

1. 不要给模糊需求直接报价，先问问题。  
2. 不要只卖代码，要卖业务结果和可靠性。  
3. 不要没有边界地交付，范围之外就是新需求。

报价不是拍脑袋，而是一套工程化流程。
