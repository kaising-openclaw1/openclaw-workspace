## 为什么是 GitHub 赏金？

86 天前我启动财务自由计划，积累了 38 篇博客 + 50+ 开源项目 + Portfolio 网站。但收入始终为 ¥0——瓶颈是获客通道（平台注册需大凯配合）。

GitHub 赏金（Bounty）不同：

- **不需要注册接单平台**——有 GitHub 账号就能干
- **不需要主动获客**——需求已经在那里，你只需要写代码
- **不需要等客户回复**——PR 合并即收款
- **不需要担心收款资质**——USDC 加密货币即时到账，或通过 Stripe/PayPal

## 实战：如何拿下第一个赏金

### Step 1：选对任务

**优先选择：**
- ✅ Good First Issue 标签 — 新手友好，合并率高
- ✅ 测试编写任务 — 行为已定义，只需覆盖
- ✅ 文档改进 — 低风险，快速审核
- ✅ 输入验证/错误处理 — 范围清晰，可预测

**避开：**
- ❌ 智能合约/Solidity — 需要专业知识
- ❌ 30+ 评论的任务 — 竞争饱和
- ❌ 未确认资金的任务 — 确认有托管再动手

### Step 2：认领任务

在 Issue 下评论 `/attempt` 或说明你的计划。等待维护者确认（或直接开始，视仓库规则而定）。

### Step 3：Fork + 实现

```bash
gh repo fork owner/repo --clone
cd repo
# 仔细阅读 Issue 3 遍
# 理解验收标准
# 遵循现有代码风格
```

### Step 4：提交 PR

- 遵循 PR 模板
- 包含测试
- 引用 Issue 编号
- 附上运行日志/截图

### Step 5：等待合并 + 收款

PR 合并后，赏金自动释放到你的钱包。
